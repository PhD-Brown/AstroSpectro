# <FontAwesomeIcon icon={faFolderOpen} /> Structure du Projet

Le projet est organisé pour faciliter la reproductibilité, l'extension scientifique et l'utilisation pratique. Voici une description des répertoires principaux et de leur rôle.

### Vue d'ensemble de l'arborescence

<div className="row">

<div className="col col--6">
  <h4><FontAwesomeIcon icon={faFolder} /> Répertoires Principaux</h4>
  <ul>
    <li><code>/src</code> - Cœur du pipeline Python.</li>
    <li><code>/notebooks</code> - Orchestration et analyse interactive.</li>
    <li><code>/data</code> - Données brutes, traitées et catalogues (ignoré par Git).</li>
    <li><code>/website</code> - Source de cette documentation.</li>
    <li><code>/reports</code> & <code>/logs</code> - Rapports de session et logs.</li>
  </ul>
</div>
<div className="col col--6">
  <h4><FontAwesomeIcon icon={faFileAlt} /> Fichiers à la Racine</h4>

    <li><code>requirements.txt</code> - Dépendances Python.</li>
    <li><code>ROADMAP.md</code> - Vision et objectifs du projet.</li>

La séparation entre `src/` (le code pur) et `notebooks/` (l'orchestration) est un choix d'architecture clé pour garantir la modularité et la testabilité du code.
:::
