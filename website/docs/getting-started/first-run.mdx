# <FontAwesomeIcon icon={faSpinner} /> Premier Lancement : De A à Z

Maintenant que votre environnement est configuré, nous allons lancer le pipeline complet en utilisant le notebook principal `00_master_pipeline.ipynb`. Ce notebook orchestre toutes les étapes, du téléchargement des données à l'entraînement du modèle.

<div className="card-demo">
  <div className="card margin-bottom--lg">
    <div className="card__header">
      <span className="step-number">1</span>
      <h3><FontAwesomeIcon icon={faDownload} /> Télécharger un premier lot de données</h3>
    </div>
    <div className="card__body">
      <p>
        Si c'est votre toute première exécution, le dossier <code>data/raw/</code> est probablement vide. Le pipeline a besoin de spectres pour fonctionner.
      </p>
      <ol>
        <li><strong>Lancez Jupyter Lab ou Jupyter Notebook</strong> depuis votre terminal (où l'environnement <code>venv</code> est toujours actif).</li>
        <li>Ouvrez le notebook <code>notebooks/01_download_spectra.ipynb</code>.</li>
        <li>Ce notebook utilise le module <code>SmartDownloader</code> pour récupérer les spectres.</li>
        <li>Exécutez les cellules pour télécharger un premier lot. Un petit lot de <strong>100 à 500 spectres</strong> est suffisant pour un premier test.</li>
      </ol>
      <Admonition type="tip" title="Conseil">
        <p>
          Le downloader est "intelligent" : il garde en mémoire les plans d'observation déjà téléchargés (<code>downloaded_plans.csv</code>) pour ne pas les reprendre deux fois.
        </p>
      </Admonition>
    </div>

  <div className="card margin-bottom--lg">

    </div>
    <div className="card__body">
      Une fois que vous avez des données dans <code>data/raw/</code>, vous pouvez lancer le workflow complet.
      <ol>
        <li>Ouvrez le notebook principal : <code>notebooks/00_master_pipeline.ipynb</code>.</li>
        <li>Exécutez toutes les cellules. Vous verrez des barres de progression et des messages informatifs à chaque étape.</li>
      </ol>
      <p>Ce notebook orchestre les étapes suivantes :</p>
      <ul>
        <li>Sélection d'un lot de spectres non traités.</li>
        <li>Génération d'un catalogue local.</li>
        <li>Prétraitement et extraction de features.</li>
        <li>Entraînement et évaluation du modèle.</li>
      </ul>
    </div>

        <li><strong><code>reports/</code></strong>: Un rapport de session en JSON résumant les performances du modèle.</li>
      </ul>
      <p>Directement dans le notebook, vous verrez également apparaître un <strong>rapport de classification</strong> et une <strong>matrice de confusion</strong> pour une première analyse des performances.</p>
