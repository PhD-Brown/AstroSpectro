# <FontAwesomeIcon icon={faMapSigns} /> Feuille de Route du Projet

Cette page présente les développements prévus et les idées d'amélioration pour le pipeline **AstroSpectro**. C'est un document vivant qui évolue avec le projet. N'hésitez pas à regarder la [Roadmap](https://github.com/PhD-Brown/AstroSpectro/blob/main/ROADMAP.md) complète ici!

> N'hésitez pas à **[ouvrir une issue](https://github.com/PhD-Brown/AstroSpectro/issues/new/choose)** pour suggérer de nouvelles fonctionnalités ou discuter des priorités !

<br/>

    {/* AXE 1 : SCIENTIFIQUE */}
    <div className="col col--10 margin-bottom--lg">
      <div className="card">
        <div className="card__header">
          <h3><FontAwesomeIcon icon={faFlask} /> Axe 1 : Enrichissement Scientifique</h3>
        </div>
        <div className="card__body">
          <p>Améliorer la profondeur et la précision des informations extraites des spectres.</p>
          <ul>
            <li>[x] Extraction V1 : raies principales (Hα, Hβ, Ca II K&H, Mg b, Na D) — prominence, FWHM, EW</li>
            <li>[x] Ratios de raies et indices spectraux (TiO5, Dn4000, G4300, CaH2/3)</li>
            <li>[x] **Extraction V2 : 174 features spectrales — 9 familles physiques** (bandes moléculaires TiO/VO, Balmer étendu Hγ→H10, raies métalliques Fe/Mg/Ca/Na/Si/Ti/Cr/Ba/Sr, indices Lick/SDSS, continuum, profils de raies, couleurs synthétiques, indices composites)</li>
            <li>[x] **Cross-match Gaia DR3** (parallaxe, Teff, log g, [Fe/H], bp_rp, M_G, v_tan)</li>
            <li>[x] Mode `spectro_only=True` — exclusion des features non spectrales (ra, dec, redshift)</li>
            <li>[x] **Validation SHAP** — 97,9 % des top-30 features sont spectrales physiques</li>
            <li>[ ] Optimisation Optuna à grande échelle (~139k fichiers FITS restants)</li>
            <li>[ ] Détection d'anomalies : intersection HDBSCAN bruit ∩ top 1 % MSE autoencoder</li>
          </ul>
        </div>
      </div>
    </div>
    {/* AXE 2 : MACHINE LEARNING */}
    <div className="col col--10 margin-bottom--lg">
      <div className="card">
        <div className="card__header">
          <h3><FontAwesomeIcon icon={faHexagonNodes} /> Axe 2 : Modèles & Données</h3>
        </div>
        <div className="card__body">
          <p>Explorer des modèles plus performants et des techniques de traitement de données avancées.</p>
          <ul>
            <li>[x] **XGBoost** — modèle principal, 87 % balanced accuracy (mode spectro_only)</li>
            <li>[x] **LightGBM, CatBoost, ExtraTrees, VotingClassifier** — disponibles dans `SpectralClassifier`</li>
            <li>[x] GridSearchCV avec validation croisée stratifiée</li>
            <li>[x] Infrastructure Optuna (déployée — scalage en cours)</li>
            <li>[x] **Suivi d'expériences W&B** — 85+ runs, 121 métriques/run</li>
            <li>[x] Calibration ECE (*Expected Calibration Error*)</li>
            <li>[x] **Réduction de dimension** : PCA, UMAP, t-SNE, HDBSCAN (projet PHY-3500 + `src/dimred/` v0.3.0)</li>
            <li>[x] **Autoencoder spectral** (SpectralAutoencoder PyTorch — latent space 2D et 3D)</li>
            <li>[x] **Explicabilité SHAP** — découverte : Ca II H&K / Mg b dominent sur Balmer</li>
            <li>[+/-] SMOTE — intégré mais rôle limité depuis l'optimisation sur balanced_accuracy</li>
            <li>[ ] Caractérisation des 20 clusters HDBSCAN (croisement objets rares : variables, binaires, RGB)</li>
            <li>[ ] Extension à LAMOST DR10 (~10⁷ spectres) ou SDSS</li>
            <li>[ ] CNN 1D sur flux bruts (architecture planifiée, non implémentée)</li>
          </ul>
        </div>
      </div>
    </div>
    {/* AXE 3 : INFRASTRUCTURE */}
    <div className="col col--10 margin-bottom--lg">

            <li>[x] Parallélisation joblib (~340k spectres/h, Ryzen 9 5950X 32 fils)</li>
